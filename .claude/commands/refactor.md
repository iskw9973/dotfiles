# Refactor

指定されたコードのリファクタリングを提案・実行してください。

## 対象

ユーザーが指定したファイルまたは関数。指定がなければ現在の変更を対象とする。

## リファクタリング観点

1. **命名改善**: 意図が明確な名前に
2. **関数分割**: 長い関数を小さく
3. **重複排除**: DRY 原則に従う
4. **複雑度削減**: ネストを浅く、早期リターン
5. **型の厳密化**: any を避ける、Union 型の活用
6. **抽象化**: 適切なインターフェース、ジェネリクス

## 手順

1. 現在のコードを読んで理解
2. 問題点を特定
3. リファクタリング案を提示（Before/After）
4. ユーザーの確認後、実行

## 注意

- 動作を変えない（振る舞い保持）
- テストがあれば通ることを確認
- 一度に大きく変えすぎない
